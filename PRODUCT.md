# Product

## Register

brand

## Platform

web

## Users
Windows desktop power users, developers, and professionals who manage multiple complex tasks, window layouts, and browser sessions daily, needing an automated way to restore their state.

## Product Purpose
Docksy captures and restores Windows application states, window sizes/placements, multi-monitor positions, and browser tabs. Success means the user spends zero minutes reorganizing windows when starting work or swapping contexts.

## Positioning
Never spend another minute setting up your workspace. Docksy automates your daily environment.

## Conversion & proof
- Primary and secondary CTA: Primary: "Download Docksy for Windows (.exe)", Secondary: "View Source on GitHub"
- The line a visitor remembers after 10 seconds: Docksy restores your entire Windows desktop layout, monitors, and browser tabs in one click.
- Belief ladder:
  1. Setting up workspaces manually is a daily, frustrating waste of time.
  2. Docksy captures complete workspaces (including window placements and browser tabs) with perfect fidelity.
  3. Docksy is local-first, privacy-respecting, and fast.
  4. Docksy is open source and easy to install.
- Proof on hand: The open-source repository on GitHub, lightweight SQLite storage, and a native Win32 window management architecture.

## Brand Personality
Sleek Minimalist Light: Clean, modern, high-contrast, and professional. Uses stark off-whites, bold black ink, and a precise cobalt blue accent.

## Anti-references
Generic warm-beige SaaS websites, excessive soft shadows paired with thin borders, cluttered metric cards, and over-rounded borders (>16px).

## Design Principles
- High-Contrast Clarity: Ensure body text has strong, uncompromising contrast against the background (no washed-out gray on white).
- High-Fidelity Utility: Show real app layouts, UI previews, and clean, raw structures instead of generic iconography or illustrations.
- Focused Flow: Keep the layout clean, readable, and directed toward downloading the app and viewing the code.

## Accessibility & Inclusion
Strict adherence to WCAG AA contrast (>=4.5:1 for body copy), fully keyboard-accessible download states, and respects user prefers-reduced-motion preferences.
